Rough guide to what each is for:

- `implement` — build a feature or change from a clear ask, with review.
- `research-plan-implement` — research, then plan, then implement end-to-end.
- `review` — review a diff or change before landing it.
- `plan` — produce a plan or design without building yet.
- `research` — gather and synthesize information about a topic.
- `grill-me` — adversarially stress-test an idea, plan, or PRD with hard questions.
- `ralph` — long-running autonomous loop that keeps working until a goal is met.
- `debug` — investigate and fix the root cause of a defect.
- `audit` — systematically audit a feature, surface, or area for gaps.
- `create-workflow` — author a brand-new Smithers workflow from a description.
- `context-doctor` — diagnose and repair an agent's working context.
- `monitor-smithers` — watchdog that detects stuck, blocked, failed, or over-budget runs.

## Produce

1. `recommendedWorkflow` — the single best-fit id from the list. Prefer the
   classifier's `recommendedWorkflow` when it is sound; override it only if a
   different listed workflow clearly fits better.
2. `why` — why this workflow fits the task, naming the durable behaviour it
   provides (ordering, recovery, approval, or loops) that the task needs.
3. `alternativeWorkflows` — other listed ids that could also fit, best-first; an
   empty array if nothing else is a reasonable second choice.
