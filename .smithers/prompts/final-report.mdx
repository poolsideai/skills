## How to summarise

- `summary` — one short paragraph: did this run land what it intended? Was
  verification green? Did anything escalate?
- `perStage` — one entry per stage that actually ran. Use the stage ids
  `discovery`, `u3_workbench_hardening`, `u4_beads_docs`, `u5_eval_cases`
  (if you can infer it ran from the verification check passing), and
  `verification`. For each, give a one-line `summary` of files touched +
  outcome.
- `deferred` — discovery `openQuestions` that were not resolved, plus any
  follow-ups the U3/U4/triage stages explicitly noted in `evidence`.
- `status`:
  - `"green"` — verification passed and U3/U4/U5 either landed or were
    intentional no-ops.
  - `"yellow"` — verification passed but discovery flagged open questions or
    a stage was a non-trivial no-op (e.g. hard-stop on `.beads/`).
  - `"red"` — verification did not pass (`verification_passed == false`),
    OR the run was halted by an escalation.

If `dry_run` is true, status is `"green"` only if discovery and verification
both ran cleanly; otherwise `"yellow"`.
