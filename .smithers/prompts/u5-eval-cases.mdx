## Authoring requirements (from the plan + repo rules)

- The skill under `skills/workspace-inventory/` needs at least three eval
  cases under `skills/workspace-inventory/evals/<case-id>/`, including at
  least one with `bucket: "adversarial"`. `check_eval_cases.py` enforces
  this exactly.
- Each case must ship:
  - `prompt.md` — names the deterministic workspace artifact path the skill
    writes (under `.laguna/`), so without-skill arms can share the grading
    target.
  - `input/` — every file the model needs to see.
  - `expected/` — the gold artifact(s) the validator grades against.
  - `metadata.json` — canonical fields enforced by `check_eval_cases.py`
    (id, bucket, validator.script, validator.expected_status, suites, etc.).
- The case must be self-testing: replaying the case's frozen validator
  against its own `expected/` produces `validator.expected_status`.
- A good-failure adversarial case ships a deliberately-bad gold artifact and
  sets `validator.expected_status: "fail"`.

## How to work this iteration

1. If `previousCheck` is non-null, read its `stderr`/`stdout` and address the
   specific violations it called out — do not redo work that already passes.
2. Use the existing v0 skills (`ci-log-reducer`, `laguna-task-contract`,
   `repo-map`) for the metadata shape and case layout. Match their style.
3. Only add the cases the spec needs; do not invent additional coverage.
4. Write files in-place. Do not create cases outside
   `skills/workspace-inventory/evals/`.
