Built around contracts, not prompt folklore

Each skill should make the model's job smaller: declare inputs, constrain edits, return a predictable artifact, and point to a validator that can prove whether the work landed.

  1. 01 Shape the task into a compact contract
  2. 02 Run the narrow worker or router mode
  3. 03 Validate schema, patch, command, or eval result
  4. 04 Repair once, then escalate with evidence

Evaluation signals

Catalog metadata should come from repeatable runs, not vibes.

Activation precisionDid the skill trigger only when useful?
Validator pass rateDid the output actually pass the check?
Cost of passHow much time and context produced a verified success?
