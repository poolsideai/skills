## Findings considered and rejected

Recorded so they aren't re-audited next run:

- **"HeadlessTaskRunner missing spawn-error handler"** — REJECTED. `waitForChildSpawn`
  (`electron/tasks/headless-task-runner.ts:146`) binds both `spawn` and `error`
  handlers and rejects on spawn failure; `run()`'s `finally` block (`:727-729`)
  always kills the child. Error handling and cleanup are present. (Plan 003 adds
  tests to *lock in* this correct behavior, but it is not a bug.)
- **"Unawaited `Promise.all` in task controller swallows errors / races"** —
  REJECTED. `src/features/tasks/task-manager-controller.svelte.ts:252,288,299`
  use `void Promise.all([...])`, but `refreshTasks` and `loadProjects` each have
  their own try/catch setting `taskError`/`projectsError`, and `refreshTasks`
  discards stale responses via `latestRefreshRequestId` versioning (`:529,581`).
  The fire-and-forget is intentional and safe.
- **"navigation-policy.ts has no test coverage"** — REJECTED. False:
  `tests/electron/main-dev-navigation.test.ts` directly tests
  `classifyRendererNavigation`, `applyWillNavigateNavigationPolicy`, and
  `applyWindowOpenNavigationPolicy`. (Edge-case breadth could grow, but the
  claim of zero coverage is wrong.)
- **"`{@html}` / DOMPurify config unsafe"** — REJECTED. Every `{@html}` sink
  traces to DOMPurify-sanitized output (`renderMarkdown`, `src/lib/utils/markdown.ts:54`),
  highlight.js-escaped HTML, or a `sandbox="allow-scripts"` iframe
  (`artifact-viewer-document-content.svelte:547`). Sanitization is sound.
- **"`app:openPath` / `showItemInFolder` info disclosure"** — FOLDED into Plan
  004. It is the same class of issue (unscoped renderer-supplied path); the
  `shell.*` reveal-in-Finder/open-in-default-app methods inherently take a path
  and are by-design. The real fix is the root-allowlist work in 004.

## Lower-priority items noted but not planned (MED–LOW, "investigate")

Surfaced for completeness; no plan written this pass:

- **Feedback trajectory PII**: `electron/ipc/register-handlers.ts:181` accepts
  an opt-in `currentTrajectoryJson` up to 5 MB with no content redaction before
  it leaves the machine. Gated behind explicit user toggles, so consented — but
  the consent isn't informed about token/PII content in the trajectory. Worth a
  redaction-or-preview design if feedback volume grows.
- **DB incremental migrations without a transaction/version gate**:
  `electron/db/database.ts` `ensureIncrementalSchemaColumns` runs
  `addColumnIfMissing` per column without a wrapping transaction or schema
  version. SQLite DDL is individually atomic so this is low-risk; the
  multi-instance concern is plausible but unverified. Investigate before any
  larger schema change.
