## Why this matters

`pnpm audit --prod` reports 11 advisories (10 moderate, 1 low). Every one of
them is the single transitive package `hono`, reached through
`@modelcontextprotocol/sdk`. The repo currently pins `hono` to `4.12.14` via a
`pnpm.overrides` entry; the advisories are all fixed in `hono >= 4.12.18`.
These are server-side issues (JWT claim validation, cache `Vary` handling,
body-limit bypass, IPv6 deny-rule bypass, JSX/CSS injection in SSR) with low
real-world relevance to a desktop client that uses the MCP SDK as a *client*,
not as an exposed HTTP server — so this is hygiene, not an emergency. But the
fix is a one-line version bump plus a lockfile update, and it removes the
entire current audit signal so future real advisories aren't buried in noise.

## Steps

### Step 1: Bump the override

In `package.json`, change the single line inside `pnpm.overrides`:

```json
"hono": "4.12.14",
```

to:

```json
"hono": "4.12.18",
```

(If `pnpm audit` in Step 3 still reports a `hono` advisory at `4.12.18`, bump to
the lowest version the advisory's "Patched versions" field names — see STOP
conditions. Prefer the smallest version that clears all `hono` advisories.)

**Verify**: `grep '"hono"' package.json` → shows `"hono": "4.12.18",`

### Step 2: Relock

Run `pnpm install`.

**Verify**: `pnpm install` → exit 0. Then
`grep -A1 "^  hono@" pnpm-lock.yaml | head` shows a `hono@4.12.18` (or higher)
resolution, and `git diff --stat pnpm-lock.yaml` shows the lockfile changed.

### Step 3: Re-audit

Run `pnpm audit --prod`.

**Verify**: the output no longer lists any advisory whose `Package` is `hono`.
(Advisories for *other* packages, if any newly surface, are out of scope for
this plan — note them in your report but do not fix them here.)

### Step 4: Lockfile gate + typecheck + build sanity

Run, in order: `pnpm check:lockfile-release-age`, `pnpm check`,
`pnpm build:electron`.

**Verify**: all three exit 0.

Stop and report back (do not improvise) if:

- The `package.json` overrides block doesn't match the "Current state" excerpt
  (the manifest drifted since this plan was written).
- `pnpm install` fails with a peer-dependency or resolution error involving

  the repo's allowed lockfile-release-age window; report the script's message.
